## 二、商业模式

### 2.1 运营商模式

| 环节 | 类比 | 你的做法 |
|------|------|---------|
| 手机 | 免费送 | 免费帮企业搭 agent 工作台 |
| SIM卡 | 绑定运营商 | API Key 只指向你的中转站 |
| 话费 | 用了才收 | 按 token 计费，加价 3-5 倍 |
| 换运营商 | 成本高 | 换中转站 = 工作台重新搭建 |

### 2.2 锁定机制

**客户的 agent 工作台（方策系统/Dify/FastGPT）中**：
- `ONEAPI_BASE_URL` 硬编码为你的中转站地址
- `ONEAPI_API_KEY` 是你分配的专属令牌
- 客户**没有**管理后台权限修改这些配置
- 客户**没有**渠道管理、Key 管理入口
- 客户能看到的只有：用量概览、消费记录、模型库（只读）

**安全约束**：
- 客户端面板不暴露任何源头 API Key
- 客户端面板不提供修改 API 地址的入口
- 客户端面板不提供自行添加渠道/Key 的功能
- 所有流量必须经过中转站，无法绕过

### 2.3 定价策略

| 策略 | 说明 |
|------|------|
| **成本加成 3-5 倍** | DeepSeek 源头 ¥1 → 你卖 ¥3-5；Claude 源头 ¥21 → 你卖 ¥60-100 |
| **套餐制** | 基础版 ¥800/月（200万token）、专业版 ¥3000/月（1000万token） |
| **按量付费** | 充值模式，用多少扣多少 |
| **捆绑销售** | 方策系统免费 + AI额度收费（工作台免费，话费收钱） |

## 三、核心功能

### 3.1 管理员后台（你用的）

| 模块 | 功能 |
|------|------|
| **渠道管理** | 添加/编辑/删除源头 API 渠道，多 Key 轮询，优先级，故障切换 |
| **用户管理** | 创建企业客户账号，分配用户组，设置倍率 |
| **令牌管理** | 为每个客户生成专属 Key，设置配额/过期/IP白名单 |
| **模型管理** | 模型上下架，模型映射，模型定价 |
| **日志审计** | 全链路请求日志（不存对话内容，只存元数据） |
| **数据面板** | 总收入/成本/利润/请求量/错误率 |
| **告警配置** | 配额 80%/100% 告警，渠道故障告警，飞书推送 |

### 3.2 客户面板（企业客户看到的 — 参考 AI火宝 UI）

| 模块 | 功能 | 安全约束 |
|------|------|---------|
| **用量概览** | 余额、已消费、用量趋势 | 只读，看得见自己的 |
| **API Keys** | 查看自己的 Key（脱敏显示） | **不能新增/修改/删除** |
| **充值** | 在线充值/套餐购买 | — |
| **账单** | 消费记录、按模型/日期筛选 | 只看自己的 |
| **模型库** | 可用模型列表（卡片式展示） | 只读，管理员控制可见范围 |
| **演武场** | 在线测试模型（Playground） | 消耗额度 |

**客户面板不包含**：
- ❌ 渠道管理（看不到源头 Key）
- ❌ 修改 API 地址
- ❌ 添加自己的渠道/Key
- ❌ 用户组/倍率设置
- ❌ 其他客户的数据

### 3.3 API 网关（核心转发层）

| 能力 | 说明 |
|------|------|
| **OpenAI 兼容** | 100% 兼容 Chat/Completions/Embeddings/Images API |
| **流式 SSE** | 支持 stream=true 实时输出 |
| **多模态** | Vision（图片输入）、DALL-E/CogView（图片生成） |
| **模型路由** | 客户调 gpt-4o → 实际转发 deepseek-chat（灵活映射） |
| **负载均衡** | 同一渠道多 Key 轮询，加权分配 |
| **故障转移** | 主渠道超时/报错 → 自动切备用渠道 |
| **限流熔断** | 每秒/每分钟请求上限，超限返回 429 |
| **配额硬限** | 超额直接拒绝，不欠费 |

## 四、安全设计（重中之重）

### 4.1 API Key 安全

| 措施 | 说明 |
|------|------|
| 源头 Key 加密存储 | 数据库中 AES 加密，不明文 |
| 客户 Key 脱敏显示 | 面板只显示 `sk-xxx****xxxx` |
| Key 不可导出 | 客户面板没有"复制全部 Key"功能 |
| Key 绑定 IP | 可选，限制来源 IP（防 Key 泄露被盗用） |

### 4.2 防攻击

| 攻击类型 | 防护措施 |
|---------|---------|
| DDoS | Nginx 限流 + Cloudflare CDN |
| 暴力猜 Key | 连续错误 Key 封 IP 5 分钟 |
| SQL 注入 | ORM 参数化查询（NewAPI 用 GORM） |
| XSS | 前端框架自动转义 + CSP 头 |
| SSRF | 转发目标白名单（只转发到已知厂商域名） |
| 重放攻击 | 请求 ID 去重 + timestamp 校验 |

### 4.3 数据安全

| 措施 | 说明 |
|------|------|
| 不存对话内容 | 日志只记：令牌ID/模型/token数/耗时/状态码 |
| HTTPS 强制 | 全链路 TLS，不接受 HTTP |
| 数据库加密 | MySQL 启用 TDE，备份加密 |
| 审计日志 | 管理员操作（改配额/改价格/删Key）全部记录 |

### 4.4 客户隔离

| 措施 | 说明 |
|------|------|
| 令牌隔离 | 每客户独立 Key，互相看不到 |
| 数据隔离 | 客户只能查自己的用量/账单 |
| 配额隔离 | A 客户用完不影响 B 客户 |
| 限流隔离 | 每客户独立限流桶，互不干扰 |

## 五、技术方案

### 5.1 基于 NewAPI 二次开发

> GitHub: https://github.com/Calcium-Ion/new-api
> 基于 OneAPI，Go + React，功能最全

**需要定制的部分**：
1. 客户面板权限裁剪（去掉渠道管理、Key 修改等入口）
2. 倍率/定价模块强化（支持分客户定价）
3. 品牌定制（Logo、名称、主题色）
4. 飞书告警集成
5. 对话内容日志默认关闭

### 5.2 部署架构

```
域名：api.modelhub.cn（举例）

┌─────────────────────────────────────────────────┐
│  云服务器 2C4G（阿里云/腾讯云）                     │
│                                                   │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐       │
│  │ Nginx    │  │ NewAPI   │  │ MySQL    │       │
│  │ :443     │→│ :3000    │→│ :3306    │       │
│  │ SSL+限流  │  │ Go 服务   │  │ 持久化    │       │
│  └──────────┘  └──────────┘  └──────────┘       │
│       │              │                            │
│  Cloudflare     转发到源头厂商                      │
│  CDN + WAF      OpenAI/DeepSeek/Claude/...       │
└─────────────────────────────────────────────────┘
```

### 5.3 容灾

| 措施 | 说明 |
|------|------|
| 数据库备份 | 每日自动备份到 COS |
| 渠道冗余 | 每个模型至少 2 个渠道 |
| 多 Key 轮询 | 单个 Key 被限流自动切下一个 |
| 进程守护 | systemd / Docker restart always |

## 七、与现有项目的关系

```
F:\api中转站\           ← ModelHub（本项目，基础设施层）
F:\活动公司agent\       ← 方策系统（消费端，被锁定到 ModelHub）
FastGPT / Dify          ← AI 应用平台（可选，同样锁定到 ModelHub）

关系：
  ModelHub 提供 API → 方策系统/Dify/FastGPT 消费
  企业客户通过工作台使用 AI → 产生 token 消耗 → ModelHub 计费
  企业客户无法绕过 ModelHub → 持续收入
```

## 八、风险与应对

| 风险 | 应对 |
|------|------|
| 客户发现加价太高 | 定价参考 AI火宝/API2D 市场价，不超过市场均价 |
| 源头降价 | 相应降价，保持倍率不变 |
| 客户要自建 | 强调"我们负责运维，你专注业务"，降低客户 DIY 动力 |
| API Key 泄露 | IP 白名单 + 配额硬限 + 实时告警 |
| 渠道宕机 | 多渠道冗余 + 自动故障切换 |
| 合规风险 | 不存对话内容，只存元数据 |
| 服务器被攻击 | Cloudflare WAF + Nginx 限流 + fail2ban |
